# Teams Mute Overlay 功能介绍

Teams Mute Overlay 是一个运行在 macOS 上的 Microsoft Teams 静音状态辅助工具。它提供菜单栏入口和桌面悬浮按钮，用于实时查看 Teams 会议麦克风状态，并在 Teams API 允许时快速切换静音。

项目不会通过系统麦克风状态去猜测 Teams 是否静音，而是依赖 Microsoft Teams 桌面端的本地 Third-party app API。也就是说，只有 Teams 本地 API 可用、已配对，并且能返回会议与麦克风状态时，应用才会提供可靠的静音状态和切换能力。

## 主要功能

### 1. Teams 本地 API 连接与配对

应用会连接 Teams 暴露在本机的 WebSocket API，并监听 Teams 返回的会议状态、麦克风状态、权限状态和 token 刷新事件。

当 Teams 尚未授权当前应用时，应用会进入配对状态，并可以向 Teams 发起配对探测。用户需要在 Teams 弹出的 Allow/Block 提示中允许访问。配对成功后，Teams 返回的 token 会被保存下来，后续启动时可以复用。

### 2. 实时会议与静音状态显示

应用会根据 Teams API 返回的信息识别当前状态，包括：

- Teams API 不可用或 Teams 未运行
- 已连接但未进入会议
- 需要配对
- 当前处于静音
- 当前麦克风开启
- 正在等待 Teams 确认静音切换结果
- API 或本地处理错误

这些状态会显示在菜单栏标题和悬浮按钮上。悬浮按钮使用不同颜色区分状态，例如静音、麦克风开启、同步中、错误或未就绪。

### 3. 悬浮静音按钮

应用启动后会显示一个始终置顶的圆形悬浮按钮。按钮可以拖动，并会保存上次停靠位置。

当 Teams 明确报告当前处于会议中，且状态为静音或未静音时，点击悬浮按钮会向 Teams 发送 `toggle-mute` 指令。应用不会在 API 状态不明确时盲目切换，以避免误操作。

点击切换后，应用会先进入同步状态，并等待 Teams 返回新的会议状态来确认切换是否成功。如果 Teams 在短时间内没有确认，应用会显示错误并尝试重新连接。

### 4. 菜单栏控制

应用作为 macOS 菜单栏工具运行，不占用 Dock 图标。菜单栏中提供以下常用操作：

- 查看当前 Teams API 或会议状态详情
- 重新配对 Teams API
- 断开连接并清除已保存 token
- 重置悬浮窗位置
- 开关音量 Pulse
- 开关静音讲话警告
- 调整悬浮窗大小
- 退出应用

这些设置会保存到 `UserDefaults`，下次启动时继续使用。

### 5. 静音讲话警告

项目包含一个可选的本地麦克风音量监测能力，用于发现“Teams 已静音但用户正在讲话”的场景。

当“静音讲话警告”开启，并且 Teams 当前确认为静音状态时，应用会读取本机麦克风输入的实时音量峰值。若音量持续超过阈值，悬浮按钮会显示 `SPEAKING` 和 `MIC IS OFF`，并通过缩放脉冲提醒用户。

这个功能只用于实时音量检测，不会录音，也不会把音频写入磁盘。首次启用相关能力时，macOS 可能会请求麦克风权限。

### 6. 多屏幕场景辅助

悬浮窗会保存当前位置，并支持在不同屏幕间移动。静音讲话警告触发时，如果当前悬浮窗不在鼠标所在屏幕，应用会尝试把悬浮窗移动到当前鼠标所在屏幕，方便用户在多显示器环境中及时看到提醒。

### 7. Token 安全存储

Teams API 配对成功后返回的 token 会保存到 macOS Keychain，默认服务名为 `TeamsMuteOverlay`。应用不会把 token 明文写入项目文件或普通配置文件。

菜单栏中的“断开并清除 token”可以删除已保存 token，并让应用回到需要重新配对的状态。

### 8. API 可行性验证工具

项目还提供了一个命令行验证工具 `teams-mute-spike`，用于确认当前 Teams 客户端和公司策略是否允许使用 Third-party app API。

它可以执行以下验证：

- 发起 Teams API 配对
- 监听会议状态和麦克风状态
- 发送静音切换命令
- 持续监控 Teams API 返回的会议更新
- 清除 Keychain 中保存的 token

如果 Teams 未运行、API 不可用、公司策略禁用了接口，或 Teams 无法返回必要的会议/麦克风状态，工具会明确输出当前方案不可用。

## 项目组成

- `TeamsMuteOverlayCore`：核心逻辑模块，包含 Teams API 客户端、状态转换、token 存储、Teams 进程检测、麦克风音量监测和静音讲话检测。
- `TeamsMuteOverlayApp`：macOS 菜单栏应用和 SwiftUI 悬浮窗界面。
- `teams-mute-spike`：用于验证 Teams Third-party app API 可行性的命令行工具。
- `Scripts`：本地构建和运行脚本，在 SwiftPM 受本机 Command Line Tools 问题影响时也可以直接编译运行。
- `Tests`：核心状态转换、控制器行为和静音讲话检测相关测试。

## 使用前提

使用主要功能前，需要满足以下条件：

- macOS 13 或更高版本
- 已安装并运行 Microsoft Teams 桌面端
- Teams 中启用 `Settings > Privacy > Third-party app API > Manage API`
- 加入 Teams 会议后完成 API 配对授权

如果 Teams 本地 API 被禁用或不可用，本项目不会退回到系统麦克风猜测方案，而是明确显示 API 不可用或配对不可用状态。
