# Teams Mic Control 部署与运行说明

本文档说明如何在本机准备环境、验证 Teams API 可用性，并运行 Teams Mic Control。

## 环境要求

- macOS 13 或更高版本
- Microsoft Teams 桌面端
- Swift 5.8 或更高版本
- Xcode 或 Command Line Tools

项目依赖 Teams 桌面端的本地 Third-party app API。使用前需要在 Teams 中启用：

```text
Settings > Privacy > Third-party app API > Manage API
```

## 构建与测试

在项目根目录执行：

```bash
cd /Users/alex/myProject/TeamsMuteOverlay
swift build
swift test
```

如果本机只有 Command Line Tools，且 `swift build` 报出类似 `xcrun --sdk macosx --show-sdk-platform-path` 的错误，可以安装或切换到完整 Xcode：

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

如果暂时无法使用 SwiftPM，也可以用本机 SDK 做类型检查：

```bash
SDK=/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk
swiftc -typecheck -sdk "$SDK" Sources/TeamsMuteOverlayCore/*.swift
swiftc -emit-module -module-name TeamsMuteOverlayCore -sdk "$SDK" \
  -emit-module-path /tmp/TeamsMuteOverlayCore.swiftmodule Sources/TeamsMuteOverlayCore/*.swift
swiftc -typecheck -parse-as-library -sdk "$SDK" -I /tmp Sources/TeamsMuteOverlayApp/*.swift
swiftc -typecheck -parse-as-library -sdk "$SDK" -I /tmp Sources/teams-mute-spike/main.swift
```

## 验证 Teams API 可用性

先打开 Teams 并加入一个测试会议，然后运行：

```bash
swift run teams-mute-spike --pair
```

如果 SwiftPM 被本机工具链问题阻塞，可以运行本地脚本：

```bash
chmod +x Scripts/*.sh
Scripts/run-local-spike.sh --pair
```

验证成功时应看到以下结果：

- Teams 弹出 Allow/Block 配对确认
- 命令行收到并保存 `tokenRefresh`
- 命令行能收到 `meetingUpdate.meetingState.isMuted`
- 命令行能收到 `meetingUpdate.meetingState.isInMeeting`

可以继续验证静音切换：

```bash
swift run teams-mute-spike --toggle
```

或使用脚本：

```bash
Scripts/run-local-spike.sh --toggle
```

如果 Teams API 被禁用、公司策略不允许、Teams 未返回会议/麦克风状态，当前方案不可用。

## 运行悬浮窗应用

使用 SwiftPM 运行：

```bash
swift run teams-mute-overlay
```

如果 SwiftPM 被阻塞，使用本地脚本运行：

```bash
Scripts/run-local-overlay.sh
```

运行后，应用会以 macOS 菜单栏工具形式启动，并显示一个始终置顶的圆形悬浮按钮。

## 本地脚本说明

项目提供以下脚本：

- `Scripts/build-local-spike.sh`：本地构建 API 验证工具
- `Scripts/run-local-spike.sh`：本地构建并运行 API 验证工具
- `Scripts/build-local-overlay.sh`：本地构建悬浮窗应用
- `Scripts/run-local-overlay.sh`：本地构建并运行悬浮窗应用
- `Scripts/package-dmg.sh`：构建可安装的 `.app` 和 DMG，可选执行 Developer ID 签名、公证与 stapling

这些脚本只会在 Swift 源码发生变化时重新构建。由于 macOS Keychain 访问提示和本地可执行文件身份有关，减少不必要的重建可以降低重复弹出 Keychain 授权提示的概率。

## 构建可安装 DMG

本项目可以打包为 macOS `.app`，再放入 DMG 供用户拖到 `/Applications` 安装。发布打包使用 SwiftPM release 产物，不使用 `.build-local` 下的本地调试二进制，因为本地调试二进制依赖项目目录里的动态库路径，不适合作为可移动 app 分发。

先确认本机 Xcode 工具链可用：

```bash
xcode-select -p
xcrun --sdk macosx --show-sdk-platform-path
swift build -c release --product teams-mute-overlay
```

如果 `xcrun` 报 `PlatformPath` 相关错误，需要安装或切换到完整 Xcode：

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

### 本地验证打包

没有 Developer ID 证书时，可以生成仅用于本机验证的 ad-hoc 签名包：

```bash
Scripts/package-dmg.sh
```

输出文件：

```text
dist/Teams Mic Control.app
dist/Teams Mic Control.dmg
```

这个 DMG 不适合正式分发给其他用户。

### 正式站外分发

正式分发到 Mac App Store 之外时，需要 Apple Developer Program、Developer ID Application 证书，以及 notarization 使用的 Apple ID app-specific password。

```bash
TEAM_ID="APPLE_TEAM_ID" \
APPLE_ID="developer@example.com" \
APP_SPECIFIC_PASSWORD="app-specific-password" \
SIGNING_IDENTITY="Developer ID Application: Name (APPLE_TEAM_ID)" \
BUNDLE_ID="com.example.TeamsMuteOverlay" \
VERSION="0.1.0" \
BUILD_NUMBER="1" \
NOTARIZE=1 \
Scripts/package-dmg.sh
```

脚本会执行：

- `swift build -c release --product teams-mute-overlay`
- 生成 `Teams Mic Control.app/Contents/Info.plist`
- 使用 Hardened Runtime 签名 `.app`
- 生成 `dist/Teams Mic Control.dmg`
- 调用 `xcrun notarytool submit --wait`
- 调用 `xcrun stapler staple` 和 `xcrun stapler validate`

发布前建议在另一台 Gatekeeper 开启的 Mac 上验证：

```bash
spctl --assess --type open --verbose=4 "dist/Teams Mic Control.dmg"
```

## 数据与权限

- Teams API token 保存在 macOS Keychain，服务名为 `TeamsMuteOverlay`
- 悬浮窗位置、大小、开关状态保存在 `UserDefaults`
- 开启音量 Pulse 或静音讲话警告时，macOS 可能请求麦克风权限
- 麦克风功能只读取实时音量，不录音，也不把音频写入磁盘
- 安装包内嵌 `Contents/Library/LoginItems/TeamsMuteOverlayMeetingHelper.app`
- 菜单项 `Teams 会议开始时自动启动` 开启后会注册此 Login Item；关闭后会注销
- 正式使用时请从 `/Applications/Teams Mic Control.app` 启动；如果曾经从 DMG 直接运行或注册过旧版本自动启动，打开已安装的 App 一次会刷新 Login Item 到当前安装位置
- helper 登录后静默运行，只在 Teams API 报告 `meetingState.isInMeeting == true` 时启动主 App
- 如果主 App 是 helper 因会议开始而启动的，Teams API 报告会议结束后主 App 会自动退出
- 如果主 App 是用户手动打开的，退出会议不会强制关闭主 App
- 此自动启动依赖已配对过的 Teams API token；没有 token 时不会通过系统麦克风状态猜测会议
